#编译原理
* DAY 1 两个操作数的加减乘除的实现
* DAY 2 计算解释器(混合运算，操作数不可为负数，不可为变量)
    * 涉及逆波兰式
* DAY 3为编译原理大作业，文法如下:
<pre><code>已有的文法G（L）：
<center>主程序结构
L->main { DL; SL; PL;}     
变量声明语句序列
DL->DD’
D’->;DD’| ε
D->T id
T->int | float | chars
赋值语句序列
SL->SS’
S’->;SS’| ε
S->id =E
计算表达式语句序列
E->TE’
E’->+TE’| -TE’| ε
T->FT’
T’->*FT’| /FT’| ε
F->变量 | 整常量 | 带小数点的常量 | ’字符’
输出语句序列
PL->PP’
P’->;PP’| ε
P->print id
</center>
</code></pre>
    * 词法分析器需求:
        * 要求能识别的关键字
            * 定义数据类型的：int， chars ，float 
            * 定义主程序的：main
            * 定义输出的：print
        * 要求能识别常量、变量
        * 要求能识别+ - * / = ; {} ’ 这些运算符号和界符
    * 语法分析器需求:
        * 使用词法分析种别编码表中各个运算符、变量和常量的种别编码替换掉文法G（L）中的各个运算符、变量和常量（如果有必要，其他的符号也可以用你认为方便的形式替换）
        * (追加)输出语法分析树
    * 语义分析需求:
        * 输出运行结果
        *  对在声明语句定义了的变量，填入符号表（包括变量的名称和类型），程序运行结束时，输出符号表中内容
<pre><strong>符号表的格式</strong><table><tr><td>变量名</td><td>数据类型</td><td>值</td></tr></table></pre>
